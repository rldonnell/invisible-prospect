P5 Pixel Intelligence Pipeline — System Map

Complete workflow showing how anonymous website visitors are identified, enriched, scored, and pushed to GoHighLevel for each client

📡 1. Ingestion
Audience Lab Pull (Cron)
Fetches visitor data from Audience Lab segment API for each client. Paginates through results, filters by timestamp (7-day lookback), upserts into Postgres.
app/api/cron/pull-audiencelab/route.js
⏰ Daily 5:00 AM UTC
Pixel Webhook (Real-Time)
Receives real-time POST from pixel platform or batch CSV uploads. Supports both AL native (UPPER_CASE) and normalized field names. Deduplicates on HEM SHA256 hash.
app/api/webhook/pixel/route.js
🗄️ 2. Storage
Neon Postgres
Cloud Postgres (pooled connection). Core visitors table with 40+ columns: identity, enrichment, behavioral, scoring, and GHL sync fields.
lib/schema.sql
Supporting Tables
processing_runs — logs each cron execution with tier counts.
ingestion_stats — daily webhook counters per client.
lib/schema.sql
Dedup Strategy
UNIQUE(client_key, hem_sha256). HEM hash is primary identity key — not email. Visitors with multiple emails consolidate to one record.
⚙️ 3. Processing
URL Taxonomy Classifier
Maps each visited URL to category + subcategory using per-client rule sets. 7 client taxonomies with 15–80 rules each covering procedures, services, high-intent pages, etc.
lib/taxonomies.js
Intent Scoring
Points-based model: visit frequency, research pages (×25 ea), high-intent pages (×30 ea), provider research (+15), recency bonus. Tiers: HOT (70+), High (25+), Medium (5+), Low.
lib/scoring.js → scoreIntent()
Confidence Scoring
Identity trust model: name↔email match, geography, enrichment depth, fake-name detection, bot patterns, 555-phone check. Tiers: High (70+), Medium (40+), Low.
lib/scoring.js → scoreConfidence()
Process Visitors (Cron)
Orchestrator: fetches unprocessed rows, runs taxonomy + intent + confidence scoring, writes results back. Loops through all ACTIVE_CLIENTS.
app/api/cron/process-visitors/route.js
⏰ Daily 6:00 AM UTC
🚀 4. GHL Push
GoHighLevel Sync
Pushes scored visitors (Medium+ intent AND Medium+ confidence) to GHL as contacts. Creates or updates with tags, custom fields (score, tier, interests, demographics).
app/api/cron/push-ghl/route.js
⏰ Daily 7:00 AM UTC
Per-Client GHL Keys
Each client can have its own GHL sub-account API key (GHL_API_KEY_SA_SPINE, etc.) and Location ID. Falls back to shared agency key if none set.
Tags & Custom Fields
Tags: "Visitor ID", tier (HOT/High/Medium), confidence, interest-based slugs, referrer source.
Custom fields: pixel_score, pixel_tier, pixel_source, pixel_visits, demographics, company info.
📊 5. Dashboard
Client Dashboard
Next.js dashboard at /dashboard/[client]. Password-protected per client. Shows visitor list, tier breakdown, visitor detail profiles.
app/dashboard/[client]/
Visitor Profiles
Deep-dive view at /dashboard/[client]/visitor/[id]. Shows full enrichment, page history, intent score, confidence, interests, and GHL sync status.
app/dashboard/[client]/visitor/[id]/
Reports API
JSON endpoint for pulling visitor data programmatically. Used by Cowork skills for generating Excel reports and HTML dashboards.
app/api/reports/route.js

Daily Cron Pipeline

Three sequential jobs run each morning on Vercel Cron, spaced 1 hour apart

1
Pull Audience Lab
Fetch new visitor records from each client's AL segment. 7-day lookback window with overlap.
5:00 AM UTC
2
Process Visitors
Classify URLs via taxonomy, score intent (points model), score confidence (identity trust), generate tags.
6:00 AM UTC
3
Push to GHL
Create/update contacts in GoHighLevel for Medium+ intent AND Medium+ confidence visitors. Batch of 10 per run.
7:00 AM UTC
